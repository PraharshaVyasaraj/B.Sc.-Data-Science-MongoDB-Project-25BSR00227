**How this Project is Organized**

I have divided the work into two main sections to show my progress from basic data handling to advanced database logic.

**1. File: MongoDB_Common_Problems_1_to_30.md**
This section contains 30 practical examples of everyday database management. It covers common scenarios like:
*   Building a menu system for a coffee shop.
*   Creating a personal contact book.
*   Tracking daily expenses and fitness logs.
*   Managing hospital bed availability and warehouse inventory.



**2. File: B1SET13.md**
This section dives into more complex territory with 5 advanced problems. The focus here is on protecting the data and ensuring everything runs smoothly in high-stakes situations. Key features include:
*   **Schema Validation:** Setting up "gatekeeper" rules so that only properly formatted data (like a specific firmware version) can enter the system.
*   **Transactions:** Learning how to process multiple updates at once. For example, when a student is assigned a room, the room's occupancy count must update simultaneously. If one part fails, the whole process stops to prevent errors.
